# Security policy

## Supported versions

The latest AMO release receives security fixes. Severe issues in the preceding minor release may receive a coordinated backport.

## Private reporting

Do not open a public issue for a vulnerability. Use the repository's **Security → Advisories → Report a vulnerability** form with the affected version, impact, reproduction, and suggested remediation. If private vulnerability reporting is unavailable, contact the repository owner privately through the verified contact on their GitHub profile. Maintainers aim to acknowledge within three business days, provide an initial assessment within seven, and coordinate disclosure after a fix is available.

Scope includes the extension, relay, native host, update/release pipeline, prompt-injection boundaries, permission confusion, and privacy leaks. Good-faith research that avoids accessing other people's data, service disruption, social engineering, and destructive testing is welcomed and will not be pursued under anti-circumvention law to the extent maintainers can offer that safe harbor.

Never include credentials, personal data, or raw page content in a report. GitHub's private advisory is the supported encrypted reporting channel; agree on a separate encrypted transfer before sharing large evidence files.
